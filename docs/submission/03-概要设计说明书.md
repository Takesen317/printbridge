## 2 设计目标与约束

### 2.1 设计目标

- **职责清晰**:主进程、Preload、Renderer、Web Worker 严格分层;
- **安全**:渲染进程上下文隔离、文件路径校验;
- **可测试**:核心服务与状态管理单元测试覆盖;
- **可演进**:模块解耦,新增模块可平滑接入。

### 2.2 设计约束

- 桌面应用优先 Windows(x64),兼容 macOS;
- 不依赖服务端(除可选 DeepSeek API);
- 包体积与启动时间容忍度:数百 MB 级别、5 秒内可启动;
- 启发式检查与软打样均为**近似**,需在 UI 中清晰标注。

### 3.2 进程职责

| 进程 | 职责 | 不应做 |
| --- | --- | --- |
| Main | 创建窗口、菜单、文件 I/O、IPC 处理 | 直接操作 DOM |

## 5 数据流

### 5.1 图像导入流

```
用户选择文件
    │
    ▼
[Renderer] File 选择
    │
    ▼ window.electronAPI.openFile
[Preload] ipcRenderer.invoke('dialog:openFile')
    │
    ▼
[Main] dialog.showOpenDialog → fs.readFile
    │
    ▼ return { filePath, buffer }
[Renderer] 解码为 ImageData
    │
    ▼ useProjectStore.setOriginalImage(ImageData)
Zustand store 持久化排除 imageBuffer
```

### 5.2 颜色分析流

```
ColorLab 用户点击"开始分析"
    │
    ▼
调用 analyzeColor(imageData, profile)
    │
    ├─► 5x5 网格采样(sampleImagePixels)
    │
    ├─► 若 lcms-wasm 就绪 + profile CMYK:创建 sRGB→CMYK transform
    │       └─► cmsDoTransform(每像素)
    │   否则:rgbToCmyk (LAB 中间 + 纸类型 gamuts clamp)
    │
    ├─► cmykToRgb 回算
    │
    └─► deltaE(CIE2000) 累计 → ColorAnalysis
```

### 5.3 印前检查流

```
PrintAdapter 用户点击"开始检查"
    │
    ▼
调用 checkPrintReadiness(imageData, options)
    │
    ├─► 计算估算 DPI(基于纸张尺寸)
    │       └─► 低于阈值 → low-resolution 问题
    │
    ├─► 采样色域外像素(默认 100x100 步长)
    │       └─► 比例 >10% → error, >1% → warning, else → info
    │
    ├─► 边缘方差 → 是否可能缺少出血
    │
    └─► 像素分布分析 → RGB / 灰度模式告警
汇总 problems + 评分(100 - 30*error - 10*warning)
```

### 5.4 项目保存与加载流

```
保存: projectSessionService.save
  → getProjectState (Zustand) + getColorState
  → buildProjectData({ version, imageBuffer?, settings })
  → window.electronAPI.saveProject(data)
  → Main: dialog.showSaveDialog + fs.writeFile(JSON)
加载: projectSessionService.load
  → window.electronAPI.loadProject
  → Main: dialog.showOpenDialog + fs.readFile + JSON.parse
  → Renderer: parseProjectData → applyProjectState / applyColorState
```

### 5.5 AI 顾问流

```
用户选择目标用途并点击"运行 AI 分析"
    │
    ▼ analyzeImageWithAI(imageData, targetUse)
    │
    ├─► extractColorData(imageData) → 颜色特征
    │
    ├─► 若 VITE_DEEPSEEK_API_KEY 存在:
    │       POST DeepSeek /v1/chat/completions
    │       解析响应 → AIColorAdvice(source: 'deepseek')
    │   失败/无 key:
    │       simulateAIAnalysis → AIColorAdvice(source: 'rule-based')
    │
    └─► useProjectStore.setAiAdvice(advice)
```

## 6 状态管理设计

### 6.1 Store 划分

| Store | 职责 | 持久化字段 |
| --- | --- | --- |
| useProjectStore | 项目级状态(图像、处理选项、AI 建议、视图模式、导出配置) | projectName、processingOptions、aiAdvice、lastViewMode、exportFormat、exportSource、aiTargetUse |
| useColorStore | 颜色相关(ICC profile、分析结果、软打样开关、阈值、自定义 profile、ICC 引擎状态) | softProofEnabled、warningThreshold、customProfiles |
| useThemeStore | 主题模式 | (由持久化中间件统一处理) |
| useLocaleStore | 语言(zh-CN / en-US) | (由持久化中间件统一处理) |

### 6.2 状态约束

- **不持久化 `imageBuffer`**:避免 localStorage 容量爆炸与性能问题;
- **不持久化派生数据**:分析结果、AI 建议可通过重新计算还原;
- **持久化 key 命名**:`printbridge-project`、`printbridge-color` 等,避免命名空间冲突。

### 6.3 跨 Store 同步

- 项目会话保存/加载需要同时同步 `useColorStore` 与 `useProjectStore`,通过 `projectSessionService` 协调(`getColorState` / `applyColorState`)。

## 7 接口设计

### 7.1 IPC 接口

| 通道 | 入参 | 出参 | 错误处理 |
| --- | --- | --- | --- |
| `dialog:openFile` | `{ filters? }` | `{ filePath, buffer } \| null` | 读取失败返回 null |
| `dialog:saveFile` | `Uint8Array, { extension? }` | `boolean` | 写入失败返回 false |
| `fs:readFile` | `filePath: string` | `Uint8Array` | 路径非法或读取失败抛错 |
| `image:process` | `Uint8Array, options` | `Uint8Array` | 抛错 |
| `project:load` | - | `ProjectData \| null` | 解析失败返回 null |
| `project:save` | `ProjectData` | `boolean` | 写入失败返回 false |
| `menu:<action>` | - | - | 主进程 `webContents.send` |

### 7.2 Renderer API(`window.electronAPI`)

```ts
interface ElectronAPI {
  openFile: (options?: { filters?: FileFilter[] }) => Promise<{ filePath: string; buffer: Uint8Array } | null>
  saveFile: (data: Uint8Array, options?: { extension?: string }) => Promise<boolean>
  readFile: (filePath: string) => Promise<Uint8Array>
  processImage: (buffer: Uint8Array, options: ProcessOptions) => Promise<Uint8Array>
  loadProject: () => Promise<ProjectData | null>
  saveProject: (data: ProjectData) => Promise<boolean>
  onMenuAction: (callback: (action: MenuAction) => void) => () => void
}
```

### 7.3 关键服务接口

```ts

async function initializeColorEngine(): Promise<{ success: boolean; message?: string }>

  imageData: ImageData,

// project-session

  save(): Promise<boolean>
  load(): Promise<boolean>
  reset(): void
}
```

## 8 关键算法概要

### 8.1 RGB → CMYK(LAB 中间)

- **路径**:RGB → LAB → 纸张色域 clamp → RGB → K-based CMYK;
- **色域边界**:coated / uncoated / newsprint 三类;
- **墨水限制**:根据纸张类型调整 K 上限(1.00 / 0.95 / 0.85)。

### 8.2 ΔE(CIE2000)

- 通过 chroma-js 计算:`chroma.deltaE(hex1, hex2)`。

### 8.3 5×5 网格采样

- 在图像宽高上等距取 5 行 5 列交点像素,固定 25 个采样点;
- 用于代表性颜色与色域分析。

### 8.4 印前启发式评分

```
score = max(0, 100 - errorCount*30 - warningCount*10)
canPrint = (errorCount === 0)
```

### 8.5 图像清晰度因子

```
distanceFactor = (250 / viewingDistance) ^ 1.4
resolutionFactor = (resolution / 300) ^ 0.9
sharpnessFactor = clamp(distanceFactor * resolutionFactor, 0.35, 1.7)
```

### 8.6 ICC Transform 生命周期

- 每次 `analyzeColor` 临时创建 transform,使用后立即 `deleteTransform`;
- `lcms-wasm` 状态保存在 `icc-handler` 单例中,跨调用复用。

## 9 错误处理与降级

### 9.1 错误分类

| 类别 | 触发 | 处理 |
| --- | --- | --- |
| 启动错误 | Electron 主进程异常 | 进程退出 |
| 渲染错误 | 组件抛错 | ErrorBoundary 兜底,显示错误信息与"重新加载"按钮 |
| 资源不可用 | lcms-wasm 初始化失败 | `iccEngineStatus = 'degraded'`,UI 显示 fallback 文案 |
| 远程失败 | DeepSeek 调用失败 | 自动回退到规则建议,`source: 'rule-based'` |
| 文件错误 | fs 读取失败 | Preload 兜底 catch,返回 null/false;UI 弹 message |
| 路径安全 | 路径含 `..`/`~`/`$` | 主进程拒绝,Preload 重抛 |

### 9.2 降级路径

```
真实 ICC (lcms-wasm)  ──失败──►  简化 LAB 中间转换 (rgbToCmyk)
DeepSeek API         ──失败──►  内置规则建议 (simulateAIAnalysis)
Web Worker           ──失败──►  主线程同步处理(并提示性能降级)
```

## 10 安全设计

- **进程隔离**:`contextIsolation: true`、`nodeIntegration: false`;
- **API 最小暴露**:仅暴露 `openFile`、`saveFile`、`readFile`、`processImage`、`loadProject`、`saveProject`、`onMenuAction`;
- **路径校验**:`fs:readFile` 拒绝路径包含 `..`、`~`、`$`;
- **CSP**(开发期):由 Vite + Electron 默认提供;
- **依赖审计**:仅使用 MIT/BSD/Apache 协议依赖。

---

## 11 性能设计

- **采样代替全像素**:5×5 网格 + 色域采样步长(默认 max/100),避免对百万级像素做逐像素计算;
- **Web Worker**:重计算放到 Worker,主线程保持 60fps;
- **状态分区**:多个 Zustand store 减少无关订阅;
- **持久化裁剪**:`partialize` 仅持久化轻量字段;
- **图像导出**:走 `canvas.toBlob` + `tif` 通过 `utif.encode`。

---

## 12 国际化与主题

- **i18n**:自研字典(`constants/i18n.ts`),`zh-CN` / `en-US`,`translate(locale, key, params?)` 支持占位符;
- **主题**:通过 `useThemeStore` + CSS 变量(`var(--color-*)`)切换;
- **样式规范**:Ant Design + 自定义 CSS 变量统一管理。

## 15 总体设计评审要点

1. **职责清晰**:主/Preload/Renderer/Worker 分层,无跨层调用;
2. **错误兜底**:lcms-wasm、DeepSeek、Worker 三层降级路径完备;
3. **状态边界**:project/color/theme/locale 各 store 边界清晰,持久化字段受控;
4. **测试可达**:Services 与 Store 均为纯函数或可注入依赖,易于单测;
5. **可演进**:新增模块通过 `modules.tsx` 注册即可,无需修改 App 与 Layout。
