## 1 引言

### 1.1 编写目的

本文档定义 PrintBridge 桌面应用的功能需求与非功能需求,作为概要设计与详细设计的输入,并作为测试计划与验收的依据。

### 1.2 项目背景

PrintBridge 是一个面向"数字媒体预览 → 印刷导向检查"桥接场景的桌面应用,定位为教学/学习工具,目标人群为设计入门者、学习者、轻量创作者。

### 1.3 术语与缩略语

| 术语 | 说明 |
| --- | --- |
| ICC | International Color Consortium,色彩管理国际标准 |
| CMYK | 青(Cyan)、品红(Magenta)、黄(Yellow)、黑(Key)四色印刷 |
| LAB | CIE LAB 色彩空间,设备无关色彩模型 |
| ΔE(CIE2000) | 人眼可感知的色差度量 |
| Soft Proof(软打样) | 在屏幕上模拟印刷输出效果的近似预览 |
| IPC | Electron 主进程与渲染进程间的进程间通信 |
| RBAC | 基于角色的访问控制(本项目无强账户,仅作扩展预留) |
| `.pbp` | PrintBridge Project 项目文件扩展名 |

## 2 项目概述

### 2.1 项目目标

构建一个跨平台桌面应用,使学习者能够:

1. 快速导入图像,查看代表性颜色与近似印刷输出;
2. 理解不同观察条件(光源、纸张、距离、DPI)对最终视觉的影响;
3. 借助启发式印前检查发现常见输出风险;
4. 通过 AI 顾问获取初步的 ICC 配置文件建议(可选);
5. 在知识中心学习色彩管理与印前基础知识。

### 2.2 用户特征

| 用户类型 | 特征 |
| --- | --- |
| 学生 / 学习者 | 设计、传媒、印刷专业学生,具备基础色彩概念 |
| 设计入门者 | 偶尔需要做印刷品的非专业设计师 |
| 教师 / 讲师 | 需要演示色彩管理与印前概念的授课者 |

### 2.3 运行环境

| 项 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11(x64)、macOS 12+、Linux |
| 运行时 | Electron 28 内置 Chromium |
| Node.js(开发期) | Node.js 18+ |
| 内存 | ≥ 4 GB |
| 磁盘 | ≥ 500 MB(含应用与缓存) |

### 2.4 假定与依赖

- 用户本地拥有支持的图像文件(PNG/JPG/TIFF/BMP);
- 如需 AI 顾问,用户自行提供 DeepSeek API Key;
- 应用以本地模式运行,不依赖服务端;
- 课程作品最终交付以 Windows 便携版为主。

### 3.2 模块 F1:颜色实验室(Color Lab)

#### F1.1 图像导入

| 项 | 描述 |
| --- | --- |
| 编号 | F1.1 |
| 描述 | 支持从本地文件系统选择 PNG/JPG/JPEG/TIFF/TIF/BMP 图像 |
| 触发 | 菜单"导入图像"、Ctrl+O、模块内"选择文件"按钮 |
| 输入 | 用户选择的图像文件 |
| 输出 | 加载成功的 `ImageData`,提示"图像已导入";失败时提示"图像加载失败" |
| 校验 | 文件存在、可读、格式合法;尺寸非法时给出缩放建议 |
| 优先级 | 高 |

#### F1.2 颜色分析

| 项 | 描述 |
| --- | --- |
| 编号 | F1.2 |
| 描述 | 对导入图像进行 5×5 网格采样,获取代表性颜色与 CMYK 近似转换结果 |
| 触发 | 模块内"开始分析"按钮 |
| 输入 | 已导入图像 + 当前 ICC 工作流 |
| 输出 | `ColorAnalysis` 结构(representativeColor / convertedColor / averageDeltaE / maxDeltaE / isInGamut / inGamutPercentage / sampleCount) |
| 算法 | RGB → (LCMS 或简化 LAB) → CMYK → 回算 ΔE |
| 优先级 | 高 |

#### F1.3 软打样预览

| 项 | 描述 |
| --- | --- |
| 编号 | F1.3 |
| 描述 | 在选定的纸张与光源下近似模拟印刷输出,并可对比屏幕版本 |
| 触发 | 模块内"打开软打样"开关 |
| 输入 | 已导入图像 + 处理选项 |
| 输出 | 软打样版图像预览,可与屏幕版并排或切换 |
| 备注 | 在 ICC 引擎不可用时显示提示信息(`analysis.iccFallback`) |
| 优先级 | 高 |

## 7 用例规约

### 7.1 UC-01:导入并分析图像

- **参与者**:学习者
- **前置条件**:应用已启动
- **主流程**:
  1. 用户选择菜单"导入图像"或按 Ctrl+O;
  2. 主进程弹出文件对话框,用户选择图像;
  3. 主进程读取文件并通过 IPC 返回 buffer;
  4. 渲染进程解码为 ImageData,提示"图像已导入";
  5. 用户点击"开始分析";
  6. 系统执行颜色分析,展示 `ColorAnalysis`。
- **后置条件**:图像与分析结果在状态中保留
- **异常**:读取失败 → 提示"图像加载失败";尺寸非法 → 提示并给出缩放建议

### 7.2 UC-02:运行印前检查

- **参与者**:学习者
- **前置条件**:已导入图像
- **主流程**:
  1. 用户切换到"智能印前适配";
  2. 选择目标纸张(A4/A3/A5/自定义);
  3. 点击"开始检查";
  4. 系统执行印前检查,展示摘要卡与问题清单;
  5. 用户可点击列表项尝试应用修复。
- **异常**:无图像 → 提示"请先导入图像"

### 7.3 UC-03:跨媒介对比

- **参与者**:学习者
- **前置条件**:已导入图像
- **主流程**:
  1. 用户切换到"跨媒介预览";
  2. 选择"并排对比"或"叠加对比";
  3. 调整光源、纸张、距离、DPI 等观察条件;
  4. 系统实时更新模拟印刷输出;
  5. 点击"刷新预览"重新计算。

### 7.4 UC-04:保存与加载项目

- **参与者**:学习者
- **主流程-保存**:
  1. 用户按 Ctrl+Shift+S 或选择"保存项目";
  2. 主进程弹出保存对话框,默认 `.pbp`;
  3. 主进程将 `{ version, imageBuffer?, settings }` 写入磁盘;
  4. 提示"项目已保存"。
- **主流程-加载**:
  1. 用户按 Ctrl+Shift+O 或选择"打开项目";
  2. 主进程弹出打开对话框,过滤 `.pbp`;
  3. 主进程解析 JSON 并回传;
  4. 渲染进程恢复状态,提示"项目已加载"。

### 7.5 UC-05:AI 顾问(可选)

- **主流程**:
  1. 用户选择目标用途;
