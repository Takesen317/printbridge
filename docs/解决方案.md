### 1.2 DPI 检测使用假数据

**问题**: 硬编码 `estimatedDpi = 72`

**方案A - 从图像尺寸推断 DPI**
```
原理: 打印 DPI = 图像像素宽度 / 打印宽度(英寸)
假设图像用于 A4 纸张 (210mm × 297mm)
```
**优点**: 实现简单，不需要用户输入
**缺点**: 需要知道目标打印尺寸，且不准确
**可行性**: 中

**方案B - 从图像元数据读取 DPI**
```typescript
// 尝试从 PNG tEXt chunk 或 JPEG APP markers 读取
// PNG: pHYs chunk 包含物理像素尺寸
// JPEG: APP0/1 包含 DPI 信息
```
**优点**: 准确读取实际 DPI 值
**缺点**: 
1. 大多数图像没有嵌入 DPI 信息
2. 需要解析多种图像格式
**可行性**: 中

**方案C - 让用户指定打印尺寸，自动计算 DPI**
```typescript
const calculateDpi = (imageWidth: number, printWidthMm: number) => {
  const printWidthInches = printWidthMm / 25.4
  return imageWidth / printWidthInches
}
```
**优点**: 最准确，用户明确知道输出尺寸
**缺点**: 需要 UI 让用户输入目标纸张尺寸
**可行性**: 高

**推荐方案**: 方案C - 实现纸张尺寸选择器，自动计算 DPI

### 1.3 色域超界检测是死代码

**问题**: `cmyk.c > 100` 条件永远为 false

**根因分析**: `rgbToCmyk` 已经将 CMYK 值 clamp 到 0-100

**方案A - 正确的 RGB→CMYK 出界检测**
```
原理: RGB→CMYK 转换时，如果任何 CMYK 值 > 100，说明原 RGB 颜色超出了目标 CMYK 色域
```
**实现**:
```typescript
function checkOutOfGamut(rgb: RGB): boolean {
  const k = 1 - Math.max(rgb.r/255, rgb.g/255, rgb.b/255)
  if (k === 1) return false  // 纯黑色不出界
  
  const c = (1 - rgb.r/255 - k) / (1 - k)
  const m = (1 - rgb.g/255 - k) / (1 - k)  
  const y = (1 - rgb.b/255 - k) / (1 - k)
  
  // 如果任何值超过 1（即 100%），说明出界
  return c > 1 || m > 1 || y > 1
}
```
**优点**: 逻辑正确，检测准确
**缺点**: 需要在 rgbToCmyk 转换过程中检测，而非之后
**可行性**: 高

**方案B - 采样+模拟转换+色差检测**
```
1. 采样图像中的所有像素（或每隔N个采样）
2. 对每个像素做 RGB→CMYK→RGB 往返转换
3. 计算色差 ΔE
4. 如果 ΔE > 阈值（如 5），标记为出界
```
**优点**: 考虑了整个转换链路的累积误差
**缺点**: 计算量大，需要遍历所有像素
**可行性**: 高

**推荐方案**: 方案A - 在转换过程中检测出界

### 1.4 出血检测逻辑错误

**问题**:
1. 只检查顶边
2. alpha < 255 ≠ 出血

**方案A - 正确理解出血**
```
出血定义: 图像内容超出裁切线之外的区域
检测方法: 检查图像边缘是否存在实际内容（非白色/背景色）
```
**实现**:
```typescript
function hasBleed(imageData: ImageData, bleedPixels: number): boolean {
  const { width, height, data } = imageData
  
  // 检查四边指定像素数内是否有非背景内容
  // 需要先确定背景色（通常是边缘像素的常见色）
  
  const edgePixels = new Set<string>()
  
  // 提取边缘像素颜色（排除角角落）
  for (let x = 0; x < width; x++) {
    edgePixels.add(getPixelKey(data, x, 0))  // 顶边
    edgePixels.add(getPixelKey(data, x, height-1))  // 底边
  }
  for (let y = 0; y < height; y++) {
    edgePixels.add(getPixelKey(data, 0, y))  // 左边
    edgePixels.add(getPixelKey(data, width-1, y))  // 右边
  }
  
  // 统计边缘颜色出现频率，找到背景色
  // 检查是否有内容延伸到边缘之外
}
```
**优点**: 逻辑正确
**缺点**: 实现复杂，需要背景色检测算法
**可行性**: 中

**方案B - 简化方案（推荐用于教学工具）**
```
1. 检查图像边缘是否为纯色/背景
2. 如果边缘有渐变或内容，说明可能有出血
3. 使用边缘检测算子（如 Sobel）
```
**实现**:
```typescript
function checkBleed(imageData: ImageData, bleedThreshold: number): boolean {
  const { width, height, data } = imageData
  
  // 获取边缘颜色（采样而非全部）
  const topColors = sampleEdgeColors(data, width, 0, 'horizontal')
  const bottomColors = sampleEdgeColors(data, width, height-1, 'horizontal')
  const leftColors = sampleEdgeColors(data, height, 0, 'vertical')
  const rightColors = sampleEdgeColors(data, height, width-1, 'vertical')
  
  // 检查边缘颜色是否一致（容差范围内）
  const variance = calculateColorVariance([
    ...topColors, ...bottomColors, ...leftColors, ...rightColors
  ])
  
  // 如果边缘颜色变化大，说明有内容延伸到边缘
  return variance > bleedThreshold
}
```
**优点**: 实现相对简单，适用于教学场景
**缺点**: 不够精确
**可行性**: 高

**推荐方案**: 方案B - 边缘颜色方差检测

### 1.5 色彩分析只检查中心像素

**问题**: `analyzeColor` 只分析一个像素

**方案A - 采样分析**
```
1. 将图像分成 5×5 或 10×10 的网格
2. 在每个网格区域中心采样一个像素
3. 汇总所有采样点的 ΔE、色域信息
4. 计算平均值、最大值、分布
```
**实现**:
```typescript
function analyzeColorSampling(imageData: ImageData, profile: ICCProfile, gridSize = 5): ColorAnalysis {
  const { width, height, data } = imageData
  const cellWidth = width / gridSize
  const cellHeight = height / gridSize
  
  const samples: RGB[] = []
  
  for (let row = 0; row < gridSize; row++) {
    for (let col = 0; col < gridSize; col++) {
      const cx = Math.floor(col * cellWidth + cellWidth / 2)
      const cy = Math.floor(row * cellHeight + cellHeight / 2)
      const idx = (cy * width + cx) * 4
      samples.push({
        r: data[idx],
        g: data[idx + 1],
        b: data[idx + 2]
      })
    }
  }
  
  // 对所有采样点进行分析
  // ...
}
```
**优点**: 结果更具代表性，实现简单
**缺点**: 仍然是网格采样，可能漏掉小范围高饱和度区域
**可行性**: 高

**方案B - 全像素分析**
```
遍历所有像素进行统计
```
**优点**: 最准确
**缺点**: 大图像性能差（1000×1000 = 100万次迭代）
**可行性**: 中（需要优化）

**推荐方案**: 方案A - 5×5 网格采样分析

### 2.1 toRealImageData 函数重复

**方案**: 提取到 `src/renderer/utils/image-utils.ts`

```typescript
// src/renderer/utils/image-utils.ts
export function toRealImageData(obj: unknown): ImageData | null {
  if (obj instanceof ImageData) return obj
  if (obj !== null && typeof obj === 'object' && 'width' in obj && 'height' in obj && 'data' in obj) {
    const o = obj as { width: number; height: number; data: Uint8ClampedArray }
    if (o.data instanceof Uint8ClampedArray) {
      const data = new Uint8ClampedArray(o.data)
      return new ImageData(data, o.width, o.height)
    }
  }
  return null
}

export function cloneImageData(imageData: ImageData): ImageData {
  return new ImageData(
    new Uint8ClampedArray(imageData.data),
    imageData.width,
    imageData.height,
    { colorSpace: imageData.colorSpace }
  )
}
```

**优点**: DRY 原则，修改一处即可
**可行性**: 高

### 2.2 InteractiveDemo 饱和度公式错误

**当前错误代码**:
```typescript
adjustedR = ... * 255 * adjustedSat  // 颠倒
```

**正确公式**:
```typescript
// 饱和度调整: 将颜色向灰色靠拢或远离
// saturation = 0: 完全去饱和（变成灰色）
// saturation = 1: 保持原色
// saturation > 1: 增强饱和度

const gray = 128
const factor = saturationAdjust  // 0-2 范围

adjustedR = gray + (R - gray) * factor
adjustedG = gray + (G - gray) * factor
adjustedB = gray + (B - gray) * factor
```

**优点**: 符合色彩科学
**可行性**: 高

## 五、验证方法

### 5.1 色彩分析验证
1. 导入一张颜色丰富的测试图
2. 检查分析结果是否反映图像整体而非单一颜色
3. ΔE 分布应该合理

### 5.2 出血检测验证
1. 用一张有明显出血的图像测试
2. 用一张没有出血的图像测试
3. 检查检测结果与实际是否一致

### 5.3 DPI 检测验证
1. 导入不同尺寸的图像
2. 选择不同纸张尺寸
3. 检查计算出的 DPI 是否合理
